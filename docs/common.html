class shuttle.common.model.JobConfig(dataset, source, target, config_file, business_application, wait=None, dry_run=False, tokenize=False, s3_format='parquet', sdp_protocol='rest', delta_key=None, update_mode=None, sqoop_update_key=None, sqoop_mappers=16, num_partitions=8)[source]

The JobConfig class configures a job for the Pipeline class.

Parameters

  • dataset (str) – Extract name.

  • source (str) –

    Source location. Accepted values are:

    • mssql: Data is located in SQL Server only.

    • psql: Data is located in PostgreSQL only.

    • hybrid: Data is located in both SQL Server and PostgreSQL.

    • s3: Data is staged in s3.

  • target (str) –

    Target location. Accepted values are:

    • s3: Stage data in s3.

    • ops: Write data to the ops database in SQL Server.

    • ol: Submit data to OneIngest for downstream publishing to One Lake, Snowflake, etc.

  • config_file (str) – Configuration file for the application environment.

  • business_application (str, optional) – OneStream business application. Required for target=ol and tokenize=True.

  • wait (str, optional) – Declare step dependency. Max wait is 2.5 hours.

  • dry_run (bool, optional) – Run job without writing to target location.

  • tokenize (bool, optional) – Tokenize extracted data.

  • s3_format (str, optional) – File format for s3 operations. Accepted values are parquet and csv.

  • sdp_protocol (str, optional) – Protocol for publishing data to OneIngest. Accepted values are rest and grpc. Publishing with rest requires first staging in s3, i.e., target=[s3, ol].

  • delta_key (str, optional) – Perform delta based on the maximum value of this key in the previous extract. Source query must use {delta_key} to denote replacement.

  • update_mode (str, optional) –

    Method for updating an existing table. Required for target=ops. Accepted values are:

    • overwrite: Truncate then load all.

    • append: Append all. This mode exports from s3 with Sqoop, i.e., target=[s3, ops].

    • upsert: Update or insert based on sqoop_update_key. This mode exports from s3 with Sqoop, i.e., target=[s3, ops].

  • sqoop_update_key (str, optional) – Comma delimited list of update keys. Required for update_mode=upsert.

  • sqoop_mappers (int, optional) – Sqoop parellelism.

  • num_partitions (int, optional) – Spark parallelism.
